Video Clipper MVP — Development Plan


Original artefact : https://claude.ai/public/artifacts/43f715ea-c79b-46d6-b5c2-93baf8b02e94
Project Overview
A web platform where users upload a long-form video and receive 3–5 AI-generated short clips with subtitles, ready to download. Monetized via subscription plans with a flexible credit system.
Core flow: Sign up → Choose plan → Buy credits → Upload video → AI finds viral moments → Render clips with subtitles → Download

	Credit Flow
1. User signs up → assigned to Free plan → receives 5 one-time credits
2. User subscribes to Pro/Business → chooses credit amount (e.g. Pro 20 credits/Rp20.000) → Duitku transaction
3. On successful payment → subscription activated → credits added to balance
4. Each billing cycle (monthly) → Duitku recurring transaction → credits replenished to the subscribed amount
5. User starts a job → credits deducted upfront based on estimated video duration
6. If job fails → credits refunded automatically via CreditTransaction with type REFUND
7. Unused credits do not roll over to the next month — they reset on renewal
8. User wants more credits mid-cycle → must upgrade to a higher credit option within their tier (e.g. Pro 20 → Pro 40) or upgrade tier (Pro → Business)
Duitku Integration
Duitku is an Indonesian payment gateway supporting Virtual Account (bank transfer), e-wallets (GoPay, OVO, DANA, ShopeePay, LinkAja), credit cards, and retail outlets (Alfamart, Indomaret). It uses a callback-based flow instead of webhooks.
* Subscriptions are handled manually: create a Duitku transaction for the monthly amount, on callback success activate the subscription period and grant credits
* Recurring billing: On each billing cycle, create a new Duitku transaction. If payment fails, mark subscription as PAST_DUE. If not resolved within grace period, downgrade to Free.
* All payment confirmations come via the POST /api/webhooks/duitku callback endpoint
* Signature verification: MD5(merchantCode + amount + merchantOrderId + apiKey)
* Key callback fields to handle:
   * resultCode — 00 = success, 01 = pending, 02 = failed
   * merchantOrderId — your order ID to match the transaction
   * reference — Duitku's transaction reference number
* Payment methods are fetched via Duitku's getPaymentMethod API to show available options dynamically
* Use Duitku POP (popup) for seamless checkout UX, or redirect to Duitku's hosted payment page

Notes for Claude Code Implementation
                                                                                                            * Build phase by phase. Complete each phase fully before moving to the next.
                                                                                                            * Test each processor independently before wiring them together. You can create test scripts that run a single processor with mock data.
                                                                                                            * Start with a short test video (2-3 minutes) for development. Don't test with 60-minute videos until the pipeline is stable.
                                                                                                            * The ffmpeg subtitle burning is the trickiest part. If ASS format causes issues, fall back to SRT with the simpler subtitles filter.
                                                                                                            * BullMQ job chaining: Each processor should add the next job to the queue upon completion, creating a chain: extract-audio → transcribe → detect-clips → render-clip (×N).
                                                                                                            * SSE reconnection: The frontend should auto-reconnect SSE if the connection drops (EventSource does this by default).
                                                                                                            * ElevenLabs response mapping: The Scribe v2 API returns words with type: "word" and type: "spacing" — filter out spacing entries and only keep word entries for subtitle generation.
                                                                                                            * ElevenLabs audio events: Scribe v2 can tag laughter, applause, music etc. Pass these tags to the clip detection prompt as extra context — laughter/applause moments are strong virality signals.
                                                                                                            * OpenRouter model swapping: Default is anthropic/claude-sonnet-4.6 for best clip detection quality. If you want to reduce costs, change OPENROUTER_MODEL env var to google/gemini-2.5-flash — no code changes needed. OpenRouter uses the OpenAI SDK format so the integration is identical for all models.
                                                                                                            * OpenRouter free models: For development/testing, OpenRouter offers free models (rate-limited to 200 req/day). Use openrouter/free as the model ID to avoid burning credits during development.
                                                                                                            * Duitku sandbox: Use https://sandbox.duitku.com/webapi for development. Switch to https://passport.duitku.com/webapi for production. The duitku npm package supports sandbox mode via config.
                                                                                                            * Duitku callback: Unlike Stripe webhooks, Duitku sends callbacks via HTTP POST to your callbackUrl. Always verify the signature (MD5(merchantCode + amount + merchantOrderId + apiKey)) before processing. Whitelist Duitku's IP addresses for production.
                                                                                                            * Duitku POP checkout: For a seamless UX, use Duitku POP (popup-based checkout) by including the Duitku JS script. This avoids full-page redirects and keeps users on your site.

Post-MVP Roadmap
Features to add after the MVP is stable and working well:
Migration from ffmpeg to Remotion
Post-MVP, the clip rendering pipeline should migrate from raw ffmpeg to Remotion (https://remotion.dev) for significantly better flexibility. Remotion renders videos programmatically using React components, which makes it far easier to build rich, customizable video outputs compared to ffmpeg filter chains.
Why Remotion over ffmpeg for post-MVP:
                                                                                                            * Subtitle styles become React components — trivial to customize fonts, colors, animations
                                                                                                            * Aspect ratio conversion is just a different React layout, not a complex ffmpeg filter graph
                                                                                                            * Word-by-word karaoke highlighting is straightforward with React animations
                                                                                                            * Video previews can render in the browser before committing to a full render
                                                                                                            * Templates and themes become shareable React components
                                                                                                            * Much easier for frontend developers to contribute to video rendering logic
Migration plan:
                                                                                                            * Keep ffmpeg for MVP (simple, proven, no extra dependencies)
                                                                                                            * Add @remotion/renderer and @remotion/cli to backend
                                                                                                            * Rebuild clip rendering as Remotion compositions (React components)
                                                                                                            * Use Remotion's renderMedia() for server-side rendering
                                                                                                            * ffmpeg still handles audio extraction and initial video processing
Video Aspect Ratio Conversion (via Remotion)
                                                                                                            * 9:16 vertical reformat for TikTok, Instagram Reels, YouTube Shorts
                                                                                                            * Smart crop using speaker detection or center-crop — handled as a Remotion composition layout
                                                                                                            * User toggle: "Keep original" vs "Convert to vertical (9:16)"
                                                                                                            * Preview the crop in-browser before rendering
Custom Subtitle Styles (via Remotion)
                                                                                                            * Available on Business plan (customSubtitleStyle feature flag)
                                                                                                            * User-configurable options:
                                                                                                            * Font family (Arial, Montserrat, Poppins, etc.)
                                                                                                            * Font size and weight
                                                                                                            * Text color + outline color
                                                                                                            * Background box (on/off, color, opacity)
                                                                                                            * Position (top, center, bottom)
                                                                                                            * Word-by-word highlight animation (karaoke style)
                                                                                                            * Store as SubtitlePreset model linked to User
                                                                                                            * Subtitle styles are React components — easy to add new styles as "templates"
Additional Future Features
                                                                                                            * Social media direct publishing (TikTok, Instagram, YouTube APIs)
                                                                                                            * Batch processing (upload multiple videos at once)
                                                                                                            * Team/organization accounts with shared credit pools
                                                                                                            * Custom clip duration range (user overrides 30–90s default)
                                                                                                            * Multi-language subtitle translation (via AI translation API)
                                                                                                            * Webhook/API access for power users to integrate into their workflows
                                                                                                            * Video template marketplace — users can share/sell Remotion-based clip templates
